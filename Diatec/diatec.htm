Histórico de Alterações Realizadas e Links para Download:

Versão 2.61 – 14/10/2005           Download:    Servidor

                                                                                              Admin

                                                                                              Cliente

 

Para usar a versão 2.61 é necessário atualizar todos executáveis do sistema conforme o link de download acima.

 

Novos Recursos:

 

  1. Agora é possível imprimir um termo de validação para as EDT’s (fase sumário).  No momento da validação da EDT é apresentando um campo de observação que fará parte desse termo. O campo da tabela FASE é o OBS (CHAR 255) que será enviado na UFASE.DBF para a retaguarda.

 

  1. A legenda do grid de tarefas foi corrigida para a opção ATRASADA que agora é usada corretamente quando uma tarefa está atrasada em seu inicio ou término. Além disso foi criado uma nova legenda chamada EXCEDIDO para as tarefas que já tiveram mais horas apontadas do que a quantidade prevista.

 

  1. A estimativa de andamento de fase agora pode ser realizada no momento do apontamento. Facilitando assim sua realização. Além disso pode-se tornar obrigatório essa estimativa. Esses dois itens devem ser habilitados no Diatec Admin dentro de “Regras de OS”.

 

 

Versão 2.60 – 06/10/2005           Download:    Servidor

                                                                                              Admin

                                                                                              Cliente

 

Para usar a versão 2.60 é necessário atualizar todos executáveis do sistema conforme o link de download acima. Portanto, as versões anteriores de Diatec Client não vão conseguir fazer upload/download do servidor Diatec com versão de client mais antiga.

 

Novos Recursos:

 

  1. Os feriados agora podem ser validados como hora “Fim de Semana”. Basta que o job de integração do Protheus alimente a tabela feriado.dbf que contém apenas 2 campos: DATA (CHAR 20) e DESCRICAO (CHAR 50). Para não usar esse recurso basta simplesmente não alimentar a tabela de feriados.

 

  1. Foram incluídas novas opções de classificação de comunicação de projeto. São elas:
    1. Relatório de Acompanhamento Técnico:

Campo Tipo da tabela Solicita = “RT”            

 

    1. Ata de Reunião:

Campo Tipo da tabela Solicita = “AR”

 

    1. Levantamento:

Campo Tipo da tabela Solicita = “LV”

 

  1. No relatório de comunicação de projeto foi diminuído a fonte da descrição e também tirado o negrito para um melhor aproveitamento de espaço.

 

  1. Foi criado uma nova opção dentro do DiatecAdmin para re-exportar os uploads que ainda não foram processados pelo Protheus. Essa opção está dentro da opção “Arquivo” do menu principal.

 

  1. Agora o Diatec pode usar uma segunda porta de comunicação entre os client’s e o servidor. Essa segunda porta vai usar o protocolo HTTP e tem a função de permitir o download / upload em locais bloqueados por firewall ou proxy. Uma vez configurada, a porta http será usada automaticamente quando o client não conseguir se comunicar através da porta primária (formato binário comprimido). A configuração a ser feita, tanto no servidor quanto no client, é no arquivo diatec.ini no seguinte formato:

 

portahttp = X          

 

No client essa configuração não é obrigatória, bastando o analista informar o numero da porta no momento do upload ou download. Já no servidor é obrigatório configurar o número da portahttp caso se deseje habilitar essa funcionalidade. É recomendado o uso da porta 80 para essa função caso não exista nenhum servidor WEB associado ao endereço público da ARM. Utilizando a porta 80 no formato http os firewall’s e proxies vão tratar a comunicação entre o Diatec Client e Server como se você um acesso à uma página de WEB.

 

  1. No grid de tarefas/fases de projeto, as fases EDT’s (ou sumário) agora contém também a quantidade de horas realizadas (e não apenas previstas).

 

  1. Agora é possível Validar ou Concluir uma fase EDT desde que seja habilitado essa opção no DiatecAdmin (opção “Validar EDT’s”  de “Regras para OS”). Ao validar ou concluir uma fase EDT’s todas suas fases descendentes serão também validadas/concluídas.

 

  1. Foi melhorado a validação on-server de upload. Algumas situações não estavam sendo tratadas com todo rigor possível, permitindo que o analista conseguisse fazer upload de OS repetida ou com conflito de horário. Essa brecha na validação só ocorria quando o analista fazia uploads seguidos em um curto espaço de tempo.
